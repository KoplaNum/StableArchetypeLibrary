# Contributing

Thanks for your interest in contributing.

## Getting Started

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make focused changes with clear commit messages.
4. Run local build/tests relevant to your change.
5. Open a pull request.

## Development Notes

- Read `README.md` and `BUILDING.md` first.
- Keep local/private paths out of tracked files.
- Do not commit secrets, credentials, or local machine config.

## Pull Request Guidelines

- Explain the problem and why the change is needed.
- Keep PRs small and reviewable when possible.
- Update documentation for behavior/build changes.
- Include verification steps (what you ran and results).

## Commit Style

Use short, descriptive commit messages in imperative style, for example:

- `Add issue and PR templates`
- `Fix OpenCL-GL cleanup ordering`

## Reporting Bugs

Please use the bug report issue template and include:

- steps to reproduce
- expected behavior
- actual behavior
- environment details
